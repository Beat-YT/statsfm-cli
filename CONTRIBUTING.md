# Contributing to statsfm-cli

## Before You Start

1. Open an issue or discussion before large changes
2. Fork the repo and create a feature branch from `main`

## Making Changes

- Keep PRs focused — one feature or fix per PR
- Update `SKILL.md` if you change CLI behavior (flags, commands, output format)
- Test your changes: run the affected commands manually before submitting
- Use clear commit messages (`feat:`, `fix:`, `docs:`, `refactor:`)

## Code Style

- Python 3.10+
- stdlib only — no external dependencies
- Follow existing patterns in `scripts/statsfm.py`

## Pull Requests

- PR against `main`
- Describe what changed and why
- Include example output if the change affects CLI behavior
